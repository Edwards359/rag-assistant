Машинное обучение - это раздел искусственного интеллекта, который позволяет

основных типа: обучение с учителем (supervised learning), обучение без учителя

### Пример 2: Технический вопрос

**Запрос:**
```
Как работают трансформеры в NLP?
```

**Ответ:**
```
Трансформеры - это архитектура нейронных сетей, представленная в статье 
"Attention is All You Need" в 2017 году. Они используют механизм самовнимания 
(self-attention) для обработки последовательностей данных. Ключевые особенности:

1. Механизм внимания позволяет модели фокусироваться на релевантных частях входа
2. Параллельная обработка (в отличие от рекуррентных сетей)
3. Позиционное кодирование для сохранения информации о порядке

Трансформеры стали основой для моделей типа BERT, GPT и T5.

Источник: OpenAI API (gpt-4o-mini)
Использовано документов: 3
```

### Пример 3: Вопрос про RAG

**Запрос:**
```
Что такое RAG и как он работает?
```

**Ответ:**
```
RAG (Retrieval-Augmented Generation) - это подход, который комбинирует извлечение 
информации из базы знаний с генерацией текста языковыми моделями. 

Процесс работы RAG:
1. Извлечение: Система находит релевантные документы из базы знаний
2. Контекстуализация: Найденные документы используются как контекст
3. Генерация: Языковая модель генерирует ответ на основе контекста

Преимущества RAG:
- Модель может отвечать на основе актуальной информации без переобучения
- Снижается риск галлюцинаций (выдумывания фактов)
- Можно обновлять базу знаний без обучения модели заново

Источник: OpenAI API (gpt-4o-mini)
Использовано документов: 3
```

### Пример 2: Технический вопрос

**Запрос:**
```
Объясни векторные embeddings
```

**Ответ:**
```
Векторные представления слов (embeddings) - это способ представления слов в виде 
плотных векторов фиксированной размерности. 

Ранние методы (Word2Vec, GloVe) создавали статические embeddings. Современные 
модели создают контекстуальные embeddings, где представление слова зависит от 
контекста использования.

Embeddings используются для измерения семантического сходства между словами 
и текстами.

🤖 Источник: GigaChat
Использовано документов: 3
```

## 💡 Практические советы

### 1. Эффективное использование кеша

```python
# Хорошо - формулируйте похожие вопросы одинаково
"Что такое RAG?"
"что такое rag?"  # Будет найдено в кеше (нормализация)

# Плохо - разные формулировки одного вопроса
"Что такое RAG?"
"Расскажи про RAG"
"Объясни RAG"  # Каждый вызовет API
```

### 2. Оптимизация запросов

```python
# Хорошо - конкретный вопрос
"Как работают трансформеры в NLP?"

# Плохо - слишком общий вопрос
"Расскажи про нейронные сети"  # Ответ будет слишком широким
```

### 3. Проверка статистики

```bash
# Регулярно проверяйте статистику
>>> stats

# Очищайте кеш при необходимости
>>> clear
⚠️  Вы уверены, что хотите очистить кеш? (yes/no): yes
✅ Кеш очищен
```
