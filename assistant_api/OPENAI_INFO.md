# 🔵 OpenAI RAG Ассистент - Важная информация

## 🎯 Архитектура системы

### Компоненты:

1. **OpenAI API** - генерация ответов (GPT-4o-mini)
2. **OpenAI Embeddings** - векторизация текста (text-embedding-3-small)
3. **ChromaDB** - векторная база данных
4. **SQLite** - кеш ответов
5. **RAGAS** - оценка качества RAG системы

## 📊 Используемые модели

### 1. LLM для генерации ответов:
- **Модель:** `gpt-4o-mini`
- **Контекст:** До 128K токенов
- **Temperature:** 0.3 (низкая для точности)
- **Max tokens:** 500
- **Стоимость:** ~$0.15 за 1M input токенов, ~$0.60 за 1M output токенов

### 2. Embeddings:
- **Модель:** `text-embedding-3-small`
- **Размерность:** 1536
- **Стоимость:** ~$0.02 за 1M токенов
- **Качество:** Отличное для семантического поиска

## 📈 Метрики качества (RAGAS)

### Доступные метрики:

1. **Faithfulness (Точность)**
   - Измеряет соответствие ответа контексту
   - Диапазон: 0.0 - 1.0
   - 1.0 = ответ полностью основан на контексте

2. **Context Precision (Точность контекста)**
   - Измеряет качество извлечённого контекста
   - Диапазон: 0.0 - 1.0
   - 1.0 = идеальный контекст для ответа

3. **Context Recall (Полнота контекста)**
   - Проверяет, весь ли нужный контекст извлечён
   - Диапазон: 0.0 - 1.0

4. **Answer Relevancy (Релевантность ответа)**
   - Насколько ответ соответствует вопросу
   - Диапазон: 0.0 - 1.0

### Запуск оценки:
```powershell
python evaluate_ragas.py
```

**Время выполнения:** 1-2 минуты (RAGAS использует OpenAI для оценки)

## 💾 Кеширование

### Механизм:
- **База:** SQLite (`api_rag_cache.db`)
- **Хеширование:** SHA-256 нормализованных запросов
- **Нормализация:** lowercase + удаление лишних пробелов

### Преимущества:
- ⚡ Мгновенные ответы на повторные запросы (<100ms)
- 💰 Экономия на API вызовах
- 📊 Статистика использования

### Очистка кеша:
```python
# В консольном интерфейсе
>>> clear
```

## 🗄️ Векторная база данных

### ChromaDB настройки:
- **Коллекция:** `api_rag_collection`
- **Метрика:** Cosine similarity
- **Директория:** `./chroma_db/`

### Chunking стратегия:
- **Chunk size:** 500 символов
- **Overlap:** 100 символов
- **Метод:** По абзацам → по предложениям
- **Минимум:** 50 символов

### Поиск:
- **Top K:** 3 документа
- **Метод:** Векторный поиск (embeddings)

## 📈 Оптимизация затрат

### Рекомендации:

1. **Используйте кеш максимально:**
   - Похожие вопросы формулируйте одинаково
   - Кеш учитывает нормализацию запросов

2. **Оптимизируйте chunk size:**
   - Меньше чанков = меньше embeddings
   - Но слишком большие чанки снижают точность

3. **Используйте gpt-4o-mini вместо gpt-4:**
   - В 60 раз дешевле
   - Достаточно для большинства задач

4. **Ограничьте max_tokens:**
   - 500 токенов достаточно для большинства ответов
   - Меньше токенов = меньше стоимость

## 🔧 Troubleshooting

### "OPENAI_API_KEY не установлен"
**Решение:** 
```powershell
# Создайте .env файл:
echo "OPENAI_API_KEY=sk-your-key" > .env
```

### "RateLimitError: Rate limit exceeded"
**Решение:**
- Подождите 1 минуту
- Проверьте лимиты на https://platform.openai.com/account/limits
- Используйте кеш для повторных запросов

### "InsufficientQuota: You exceeded your current quota"
**Решение:**
- Пополните баланс на https://platform.openai.com/account/billing
- Проверьте usage на https://platform.openai.com/usage

### Медленные ответы
**Решение:**
- Используйте кеш (команда `stats` покажет количество закешированных запросов)
- Уменьшите `top_k` в поиске (по умолчанию 3)
- Сократите `max_tokens` в генерации
